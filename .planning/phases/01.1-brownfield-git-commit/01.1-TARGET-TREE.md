## Root Retention Rationale

| Root Item | Why Keep At Root |
|---|---|
| `.planning/` | GSD 工具默认路径，作为任务真相源需稳定 |
| `docs/` | 文档统一入口，且 AIDP 在该层冻结 |
| `cmd/` + `internal/` | Go 工程标准分层，入口与实现分离 |
| `workflow-packages/` + `blueprint/` | 生产系统核心资产根目录，不应深埋 |
| `tests/` + `fixtures/` | 验证与样例资产，需清晰可发现 |
| `AGENTS.md` | agent 启动入口必须可直接发现 |

## Domain-Oriented Subdivision Rules

- 在技术层目录内部按领域细分：例如 `blueprint/L2/wxt/manifest`、`workflow-packages/wxt-manifest`。
- capability 资产不与主域资产混放在同一层级的主入口位置，保持 attach-only 语义清晰。
- 归档目录使用 `docs/archive/<yyyy-mm-phase>/`，按 phase 收口，避免散点归档。
