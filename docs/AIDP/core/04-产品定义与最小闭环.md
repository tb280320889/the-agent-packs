# 产品定义与最小闭环

## 当前产品定义
`the-agent-packs` 不是传统应用界面的产品，而是一个围绕增强包生产、上下文裁剪、workflow package 激活与验证闭环构建的开发基础设施仓库。

它的最终对外价值，不是让消费侧理解整个仓库，而是让消费侧 agent 能按主域和能力精准获得增强上下文。

## 当前必须跑通的闭环
1. Blueprint 以 Markdown + frontmatter 表达节点知识
2. 编译器将节点编译到 SQLite 索引
3. 路由查询层按约束筛选候选
4. Bundle 构建最小上下文包
5. Activation 组织 package、validators 与 handoff
6. Validator 输出校验结果
7. 结果以 activation result / handoff 形式返回

## 双闭环模型

### 内部生产闭环
领域知识治理 -> Blueprint / Registry 建模 -> 编译与索引 -> 路由筛选 -> context bundle 构建 -> activation / validator -> 形成可交付增强 pack 语义

### 外部消费闭环
消费请求 -> 识别主域与 capability -> 获取目标增强上下文 -> 执行任务 -> 返回结果 / handoff

### 两类闭环的关系
内部生产闭环负责“生产正确的增强 pack”，外部消费闭环负责“消费正确的增强 pack”。

开发本项目的开发 agent 主要面向内部生产闭环工作；消费侧 agent 主要面向外部消费闭环获得结果。

## 当前版本的产品重点
- 不是扩大用户可见功能面
- 而是稳定系统骨架：分层索引、包治理、知识吸收、上下文交付契约、运行态维护、准入与验证

## 当前不追求的内容
- 一次性做完所有未来领域线
- 大规模平台化抽象
- 替换全部底层实现
- 将所有旧文档无差别迁入新入口而不做语义收敛
