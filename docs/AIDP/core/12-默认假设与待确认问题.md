# 默认假设与待确认问题

## 默认假设
- 当前第一主域仍以 `WXT` 为样板主域
- 当前最紧迫目标是校准增强包生产者系统语义与角色边界，而不是立即扩大全部实现面
- 第二领域线尚未冻结为正式准入范围
- GSD 仍继续作为唯一任务真相源
- 旧 `docs/改造计划v1/` 中的阶段结论仍有效，但默认通过 AIDP 语义层消费
- 消费侧需要的默认交付面应是目标域精简上下文，而不是完整 AIDP 正文

## 待确认问题
- 第二领域线的优先候选是否已明确
- 后续是否需要把 AIDP 进一步映射为 `.planning/` 下完整正式工件
- 消费侧精简交付面最终以何种形式固化：独立文档、registry/bundle 契约、MCP 输出协议，还是多者组合
- 如何系统化验证“最小但不遗漏”在不同主域上的成立
- 何时可以正式通过退役检查表，删除 `docs/改造计划v1/`

## 阻塞级问题
- 当前无必须先回答才能继续维护 AIDP 的阻塞级问题

## 处理规则
- 若默认假设被确认或推翻，必须同步更新 `runtime/01-默认假设账本.md`
- 若待确认问题影响范围、规则或验收，必须补充到决策日志与变更摘要
