# 非功能要求

## NFR-01 可维护性
文档入口、运行态工件和 GSD 映射必须稳定，避免新 agent 只能靠旧对话和零散文档恢复上下文。

## NFR-02 可扩展性
新增主域或 capability 时，不应依赖新增硬编码分支作为唯一接入方式。

## NFR-03 可验证性
关键阶段必须具备明确验证方式，至少包含快速测试命令或文档级验收判定。

对消费侧上下文交付，还必须具备样例化验证口径，至少能用 `WXT -> wxt-manifest` 样板说明什么叫“最小但完整”。

## NFR-04 一致性
项目语义、假设、决策、变更与验证记录之间不能相互冲突；冲突时必须用变更摘要显式说明。

消费契约、路由规则、上下文裁剪规则与 MCP 暴露口径之间也不得相互冲突。

## NFR-06 可追踪性
对消费侧交付的上下文，应能追踪其来源、裁剪依据、附挂 capability 依据及验证口径。

## NFR-07 域隔离性
主域上下文交付必须具备域隔离意识，避免把无关主域知识、内部治理正文或无关 capability 混入默认输出。

## NFR-05 安全性
涉及外部输入路径、MCP JSON 输入、外部资料吸收时，应保持边界意识，避免无约束读取与过大输入风险。
