# 业务规则与关键对象

## 关键对象
- `Pack Producer`：生产和治理增强 pack 的角色与系统
- `Pack Consumer Agent`：消费增强 pack 的下游 agent
- `Blueprint Node`：定义路由节点、层级、可见性和激活模式
- `Registry PackageEntry`：定义 package 身份、领域归属、依赖、validator 与 artifact 映射
- `Domain Knowledge Slice`：按主域或 capability 裁剪后的知识切片
- `Activation Request`：请求入口对象
- `Routing Result`：路由输出对象
- `Context Bundle`：最小上下文包
- `Context Delivery Contract`：定义对消费侧上下文交付的最小性、完整性、边界性与可验证性要求
- `Artifact`：执行与验证过程中承载的结构化交付对象
- `Validation Plan`：校验计划
- `Validator Result`：校验执行结果
- `Activation Result`：最终执行结果
- `Handoff Bundle`：交接对象

## 核心规则

### BR-01：总编排先于主域编排
任何路由都应先决定当前主域候选空间，再进入域内选择。

### BR-02：主域编排先于 capability 挂接
capability 只有在主域已确认后才可进入可见范围。

### BR-03：capability 默认 attach-only
`security`、`release`、`auth`、`payment` 等横线能力，默认不得在第一轮全局主竞争中直接出现。

### BR-04：注册表是真相源
package 的身份、归属、依赖、validator/artifact 映射必须以注册表为准，不能仅靠文件名或硬编码猜测。

### BR-05：最小上下文优先
agent 默认消费最小上下文包，不以自由全文深读作为常规路径。

### BR-05A：最小上下文是正式交付契约
最小上下文不仅是内部实现原则，也是生产侧对消费侧的正式交付契约；不得用全量灌入替代精确交付。

### BR-05B：最小上下文必须同时完整
上下文最小不等于信息残缺。对当前目标主域和当前任务成立的关键规则、关键约束和关键流程不得遗漏。

### BR-06：外部资料必须经过治理管道
skill、MCP、llm.txt、外部最佳实践等不得直接原样塞入主文档，必须经过候选收集、语义归类、映射、审查和资产落位。

### BR-06A：治理产物服务于受控消费
外部资料进入项目后，不应默认以全文形式暴露给消费侧 agent，而应经过主域、能力和任务上下文裁剪后再交付。

### BR-07：顶层冻结对象不得借结构改造名义被重写
以下对象顶层语义默认不动：`Activation Request`、`Routing Result`、`Context Bundle`、`Artifact`、`Handoff Bundle`、`Validation Plan`、`Validator Result`、`Activation Result`、状态枚举与优先级、workflow package 基本模板。

### BR-08：route 必须先裁候选空间，再做评分
候选空间由 `visibility_scope + activation_mode` 决定，`aliases / triggers / target_pack` 都不能绕过合法候选空间。

### BR-09：主域、主包、横线挂接三件事都必须可解释
route 结果至少要能说明：为什么选中主域、为什么选中主 workflow package、为什么附挂这些 capability。

### BR-09A：上下文交付也必须可解释
交付给消费侧 agent 的上下文，至少要能说明：为什么包含这些知识、为什么不包含其他知识、附挂 capability 的依据是什么。

### BR-10：canonical name 与 canonical blueprint node 是正式身份约束
正式 package 必须具备稳定 canonical name；缺少稳定 `canonical_blueprint_node` 的对象，不应注册为正式可路由 package。

### BR-11：裸名 package 默认禁止
`release`、`security`、`auth`、`payment`、`manifest`、`review` 等高复用词，不得直接作为正式 package 名称。

### BR-12：alias 只能辅助检索，不能反向成为身份真相源
alias 不得覆盖 canonical name，也不得突破候选空间与激活模式约束。

### BR-13：AIDP 不等于消费侧默认交付面
`docs/AIDP/` 是生产侧语义真相源。消费侧默认交付面应是经过治理和裁剪后的增强 pack / context bundle，而不是完整 AIDP 正文。

### BR-14：开发 agent 不得混淆自身身份
开发本项目的开发 agent 必须站在生产系统建设者视角理解仓库，不得把自己默认代入最终消费侧 agent 并据此重写项目语义。

## 不允许破坏的判断
- 不允许直接改写 Activation 顶层 envelope
- 不允许 capability 与主域在同一轮无作用域过滤地竞争
- 不允许把默认假设写成已确认事实
- 不允许让运行关键上下文只留在对话里
- 不允许在没有注册表规则前继续新增裸名 package
- 不允许让 backlog / reject 资料伪装成正式资产
- 不允许把内部治理语义默认外泄为消费侧全文入口
- 不允许通过堆砌无关上下文弥补消费契约不清晰
