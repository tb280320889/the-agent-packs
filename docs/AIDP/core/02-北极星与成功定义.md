# 北极星与成功定义

## 北极星
把 `the-agent-packs` 从单样板域可运行，提升为多主域、多横线能力、多 workflow package 可持续接入的稳定增强包生产者系统，并让消费侧上下文交付、文档协议、运行态状态、GSD 映射与 agent 接力规则形成统一闭环。

## 当前版本成功定义

### 项目层成功
- 新开发 agent 能以 `docs/AIDP/` 作为统一入口，而不是依赖零散旧文档
- 关键系统判断、边界、假设、运行态状态都有稳定落点
- 项目顶层叙事明确指向“增强包生产者系统”，而不是泛化编排系统

### 系统层成功
- 路由候选集按 `global -> domain -> capability` 分层收敛
- capability 默认 attach-only，不参与第一轮主竞争
- 注册表成为 package 身份、归属与 validator/artifact 映射的真相源

### 消费层成功
- 给定目标主域时，消费侧 agent 只获得该主域和当前任务所需的上下文切片
- 消费侧默认不暴露仓库内部治理语义和无关领域知识
- 上下文交付同时满足最小性与完整性，而不是用全量喂给替代精准交付

### 协作层成功
- GSD 继续作为任务真相源
- AIDP 负责承载项目语义、运行态状态与框架映射
- 多-agent 接力时不再依赖对话中的隐式上下文
- 开发本项目的开发 agent 能清楚知道自己是在建设生产系统，而不是直接扮演消费侧 agent

### 准入层成功
- 第二领域线接入前，有明确的准入、验证和回滚边界
- 外部资料吸收必须经过治理管道后才能落为项目资产

## 不算成功的情况
- 只能靠维护者口头解释仓库怎么工作
- route 仍主要依赖平铺候选和硬编码回退
- 旧文档、AIDP 文档、运行态工件相互冲突且无人维护
- agent 无法判断什么是冻结事实、什么只是默认假设
- 开发 agent 把自己默认代入消费侧角色
- 消费侧 agent 需要读取完整 AIDP 或无关域资料才能完成任务
- 系统通过堆砌上下文来弥补消费契约不清晰
