# 全局产物映射

## 映射原则
GSD 继续承担任务真相与执行推进，AIDP 负责承载项目语义、假设、决策、变更和阶段上下文。

如需理解两者职责边界，先看 `adapters/gsd/00-协同原则.md`。

## 映射总表

| GSD 工件 | 主要来源 | 应承载内容 |
|---|---|---|
| `PROJECT.md` | `core/01`, `core/02`, `core/03`, `core/04`, `core/08`, `core/11` | 项目叙事、北极星、参与者、最小闭环、关键约束、风险摘要 |
| `REQUIREMENTS.md` | `core/05`, `core/06`, `core/09`, `core/10` | 范围、关键规则、非功能要求、验收映射 |
| `ROADMAP.md` | `core/05`, `core/10`, `protocol/06` | 当前 phase、阶段目标、输入输出、退出条件 |
| `STATE.md` | `core/12`, `runtime/01`, `runtime/02`, `runtime/03`, `runtime/04`, `runtime/05`, `runtime/06` | 已确认事实、默认假设、开放问题、最近决策、最近变化、当前阶段、验证状态 |

## 使用规则
- 不要把 AIDP 与 GSD 当成两套互不相关的系统
- GSD 记录“任务真相”，AIDP 记录“项目语义与运行状态”
- 二者冲突时，先判断是任务状态冲突还是语义结论冲突，再分别更新对应文档
- 消费侧最终获取的上下文输出，不由 GSD 文档直接充当，而应由生产系统根据 AIDP 规则裁剪生成
