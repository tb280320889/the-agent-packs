# AIDP Agent Instructions

你现在进入的是 `the-agent-packs` 仓库的 local project AIDP 文档集。

你的任务不是自由浏览所有旧文档，而是基于本地 AIDP 这个**项目业务语义真实源**，在最小上下文前提下稳定完成理解、规划、执行、验证与回写。

首先要确认自己的身份：你默认是“开发本项目的开发 / 维护角色”，正在建设和维护增强包生产系统，而不是默认扮演最终消费侧 agent。

## 读取旧文档的规则

- `docs/改造计划v1/` 仍可读取，但它是上游来源，不再是默认入口
- `.planning/codebase/` 用于 Brownfield 代码事实确认
- 如果本 AIDP 已覆盖某个事实，就不要回退到旧文档重新发明解释
- 只有当 AIDP 中明确标注“待确认”或“需追溯来源”时，才回读旧文档

## 你的工作方式

1. 先判断当前任务属于：规划、实现、验证、文档更新、调试、交接中的哪一类
2. 先读取最小必要 AIDP 文档，不做无边界全量阅读
3. 把不确定信息落入“默认假设”或“待确认问题”，不要伪装成既成事实
4. 执行后必须回写运行态工件
5. 若要继续走 GSD 工作流，先确保 local AIDP 已表达本轮业务语义，再遵循 `adapters/gsd/` 的映射规则

## 与 GSD 的关系

- GSD 是任务与执行流，不是业务语义源
- local AIDP 是业务语义真实源，通过根 `AGENTS.md` 注入给后续 GSD agent
- 后续开发 agent 应在 `.planning/`、根 `AGENTS.md`、以及最小必要的 local AIDP 文档共同指引下工作
- 新里程碑或结构性增强时，应先升级 local AIDP，再继续 GSD 工作流
- local AIDP 解释的是增强包生产系统语义，不等于消费侧 agent 默认应读取的全文知识入口

## 完成条件

只有当以下条件成立时，当前工作才算完成：

1. 当前目标已经转化为明确输出或明确阻塞
2. 已确认事实、默认假设、待确认问题已分开
3. 相关运行态文档已经回写
4. 如有验证要求，验证结果已写入 `runtime/06-验证记录.md`

## 禁止事项

- 禁止跳过 AIDP 入口直接把旧文档当主入口
- 禁止把默认假设写成已确认事实
- 禁止在未声明变更摘要时静默覆盖已有结论
- 禁止把未来平台化想法混进当前版本必做范围
- 禁止把 `.planning/` 当作业务语义真相源反向覆盖 local AIDP
