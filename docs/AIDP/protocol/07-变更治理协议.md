# 变更治理协议

## 何时算变更
- 项目目标变化
- 边界变化
- 关键规则变化
- 验收标准变化
- 默认假设被确认、推翻或替换

## 变更动作
1. 更新受影响的 `core/` 文档
2. 更新 `runtime/03-变更摘要.md`
3. 必要时更新 `runtime/02-决策日志.md`
4. 如影响执行框架，再更新 `adapters/gsd/`

## 禁止事项
- 不允许静默覆盖旧结论
- 不允许只在聊天里说明变化而不落盘
