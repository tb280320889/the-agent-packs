# 信息收集协议

## 默认来源顺序
1. `docs/AIDP/`
2. GSD 任务记录与当前运行态工件
3. `.planning/codebase/`
4. 相关代码与测试
5. `docs/改造计划v1/` 历史来源

## 原则
- 先看项目专属 AIDP，再看旧文档来源
- 先收敛当前任务所需最小上下文，再决定是否继续扩展阅读
- Brownfield 场景优先尊重现有代码事实

## 禁止事项
- 不要为了“更完整”就默认深读全部旧文档
- 不要绕过代码事实，直接把历史方案当现实实现
