# 输出生成协议

## 输出优先级
1. 先更新受影响的 `core/` 或实现
2. 再更新 `runtime/` 工件
3. 若涉及 GSD 对接，再更新 `adapters/gsd/` 或对应 GSD 文档

## 输出要求
- 只修改必要部分
- 标明是否属于事实更新、假设更新、决策更新或验证更新
- 给出不覆盖项
- 标明本次修改主要影响生产侧、消费侧还是开发侧语义
- 若影响消费侧上下文交付，必须显式说明交付契约变化

## 文档输出最低要求
- 至少说明为什么改
- 至少说明改了哪里
- 至少说明是否需要同步更新运行态工件
- 至少说明是否改变角色边界或渐进披露规则
