# 追问决策树

## 可以直接继续的情况
- `.planning/codebase/` 与 AIDP 已足够描述当前事实
- 当前任务只是文档整理、映射、实现或验证的增量推进
- 不确定项可以先落为默认假设而不影响当前输出

## 必须追问的情况
- 目标切换会显著改变范围或验收
- 涉及 breaking 决策且仓库中无冻结结论
- 涉及第二领域线正式准入，但当前没有冻结边界

## 默认策略
只要达到可生成阈值，就先产出可用首版；不要因为信息不完美而停在追问阶段。
