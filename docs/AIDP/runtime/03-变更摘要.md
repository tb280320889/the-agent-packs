# 变更摘要

## 2026-03-16 - 建立项目专属 AIDP 文档集

### 变更原因
需要把现有改造计划文档、Brownfield 代码事实和 agent 协作规则统一为新的项目默认入口，避免后续 agent 仍依赖旧目录与隐式对话上下文。

### 主要变化
- 新增 `docs/AIDP/` 项目专属 AIDP 文档集
- 新增 `core/`、`protocol/`、`runtime/`、`adapters/gsd/` 首版实例文档
- 将根目录 `AGENTS.md` 的默认阅读索引切换到 `docs/AIDP/`
- 将 `docs/改造计划v1/` 降级为历史来源而非默认入口

### 不变项
- GSD 仍是任务真相源
- 当前系统最小闭环不变
- 旧改造计划文档不删除，仍可追溯

### 后续需要继续维护的内容
- 每次重大规划或实现变化后，继续更新假设账本、决策日志、phase context 与验证记录

## 2026-03-16 - 对齐 system AIDP v2.0.1 版本治理与增强迭代机制

### 变更原因
上游 system AIDP 已引入版本治理、兼容性、增强开发与迭代机制。为保持 local project AIDP 与 system AIDP 思想一致，并支撑“先升级 local AIDP，再继续 GSD 工作流”的目标，需要把这些机制引入本地文档集。

### 主要变化
- 新增 local AIDP 版本与治理文件：`VERSION.md`、`CHANGELOG.md`、`COMPATIBILITY.md`、`GOVERNANCE.md`、`CONTRIBUTING.md`、`RELEASE-CHECKLIST.md`
- 新增 `protocol/10-增强开发与迭代协议.md`
- 新增 `runtime/08-增强开发任务启动单.md` 与 `runtime/09-迭代会话交接.md`
- 新增 `adapters/gsd/06-增强开发与变更落地.md`
- 明确 local AIDP 与上游 system AIDP 的双层版本关系

### 不变项
- GSD 仍然是任务与执行流，不被 AIDP 替代
- `.planning/` 仍然是 GSD 运行工件，不是业务语义真相源

### 当前判断
- 思想对齐：已对齐
- 旧文档退役：仍未完成最终验证

## 2026-03-16 - 校准增强包生产者系统语义与身份边界

### 变更原因
当前 AIDP 更偏“编排系统 + 内部开发入口”叙事，难以准确表达本项目作为增强包生产者系统的真实核心目标，也不足以约束消费侧上下文交付边界。

### 主要变化
- 新增 `core/00-身份视角与项目定位.md`
- 新增 `runtime/08-语义整改计划.md`
- 将 README、MANIFEST 与 core 关键文档校准为“增强包生产者系统”叙事
- 显式区分生产者、消费侧 agent、开发 agent 三类身份
- 将渐进披露和“最小但完整”提升为正式上下文交付契约
- 补充协议、GSD 协同与运行态文档，防止后续 agent 回退到旧叙事

### 不变项
- GSD 仍是任务真相源
- 当前系统 activation 主链路不变
- 第一稳定样板主域仍以 WXT 为基线

### 后续需要继续维护的内容
- 将消费侧精简交付面进一步固化为实现与验证机制
- 后续新增主域或 capability 时，持续检查是否破坏角色边界与上下文交付契约

## 2026-03-16 - 将消费契约继续映射到实现与验证口径

### 变更原因
首轮整改已明确顶层语义与角色边界，但若不继续把消费契约映射到实现与验证层，后续开发仍可能回退为抽象口号或局部理解。

### 主要变化
- 新增 `runtime/10-WXT-消费契约验证样例.md`
- 在 `core/09` 增补消费契约的可验证性、可追踪性与域隔离性要求
- 在 `core/10` 增补 `REQ-11`，把 `WXT -> wxt-manifest` 作为样板消费契约验收口径
- 在 `core/11` 增补消费契约落地风险
- 在 `adapters/gsd/06` 增补“影响消费契约时必须补样例验证口径”的落地要求

### 不变项
- 本轮仍以文档级和流程级整改为主
- 尚未宣称代码层自动化验证已经全部具备

### 后续需要继续维护的内容
- 将 `WXT` 样例继续下沉为实现检查项、测试夹具或集成验证流程
- 后续新增主域时，为每个主域建立对应消费契约样例
