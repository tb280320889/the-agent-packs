# WXT 消费契约验证样例

## 目的
本样例用于把“消费侧只获得目标域最小完整上下文”的语义要求，进一步收敛为可执行的文档级验证口径。

当前以 `WXT -> wxt-manifest` 作为第一稳定样板域。

## 样例场景
消费侧 agent 发起与 WXT manifest 开发相关的请求，希望通过 MCP 或其他受控方式获得完成当前任务所需的增强上下文。

## 应交付的上下文类型
- `WXT` 主域相关规则
- `wxt-manifest` workflow package 相关规则
- 与当前 manifest 任务直接相关的 validator / artifact / handoff 约束
- 当前任务确实需要时才附挂的横线 capability

## 不应默认交付的内容
- 无关主域的知识切片
- 与当前任务无关的 capability 正文
- 完整 `docs/AIDP/` 正文
- 仅供内部开发 agent 使用的治理细节、整改计划或运行态维护说明

## 最小但完整的判断口径

### 完整性成立
若消费侧 agent 在不回读完整 AIDP 的前提下，能够获得完成当前 WXT manifest 任务所需的关键规则、关键对象和关键约束，则视为完整性成立。

### 最小性成立
若交付内容未引入与 WXT manifest 当前任务无直接关系的主域知识、治理正文或无关 capability，则视为最小性成立。

### 可解释性成立
若可以说明：
- 为什么当前主域是 `WXT`
- 为什么当前主包是 `wxt-manifest`
- 为什么附挂或未附挂某些 capability
- 为什么包含这些上下文而不包含其他上下文
则视为可解释性成立。

## 文档级验证步骤
1. 检查 `core/04`、`core/05`、`core/06` 是否支持按主域裁剪上下文
2. 检查 `core/08` 是否要求 MCP / 暴露层保留裁剪依据
3. 检查 `core/10` 是否把 `WXT` 作为消费契约的快速验证样例
4. 检查 runtime 文档是否记录本样例的验证用途和限制

## 当前限制
- 本样例当前是文档级验证口径，不等于代码层自动化测试已经全部具备
- 后续应把本样例继续映射为实现检查项、测试夹具或集成验证流程
