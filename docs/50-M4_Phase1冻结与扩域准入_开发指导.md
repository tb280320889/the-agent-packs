# 50 M4 Phase 1 冻结与扩域准入 开发指导

## 文档类型
开发里程碑开发指导文档

## 作用域
M4 负责把首期系统正式冻结，并定义第二领域线如何准入。  
M4 不再追求“多做一个功能”，而是追求 **把第一阶段方法固定成可复制骨架**。

## 本里程碑的目标
1. 冻结 Phase 1 的公共壳层与首个切片
2. 明确什么能改、什么不能改
3. 明确第二领域线进入前的准入条件
4. 输出“复制而不是重造”的扩域说明

- 01

## 参考输入
- M0~M3 的冻结对象与回归基线

## 必须产出
1. Phase 1 冻结对象清单
2. breaking change 治理规则
3. 第二领域线准入清单
4. 可复用 skeleton 说明
5. 对外说明当前已证明的能力边界

## 明确不产出
- 第二个完整 pack 的实现
- 大范围重构
- 新的系统层对象

---

## 执行顺序

### 第一步：列冻结清单
把真正稳定下来的对象列出来，而不是笼统写“基本稳定”。

### 第二步：明确可改 / 不可改
对 envelope、route、bundle、package template、validator 基线做清楚切分。

### 第三步：定义第二领域线准入
不是谁想接就接。  
必须满足准入条件。

### 第四步：固化可复用 skeleton
把“怎么复制到第二条领域线”写成清晰步骤。

---

## 完成标准
1. Phase 1 冻结面清楚
2. 下游知道如何提出 breaking change
3. 第二领域线进入条件明确
4. 复用方法清楚，不依赖口头解释

## 本里程碑绝对不要做的事
- 顺手重写系统层
- 因为想接第二领域线就先把第一阶段再松开
- 用“以后再说”回避 breaking change 治理
