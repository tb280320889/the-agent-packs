# 运行依赖与降级策略

## 结论

如果使用方本地没有 Python，会影响当前仓库里 **M1 的本地脚本能力**（compiler/query/entry/smoke tests），但不会影响文档层面的阅读与手工消费。

## 影响范围

当前依赖 Python 的内容：
- `tools/compiler/compiler.py`
- `tools/query-mcp/query_mcp.py`
- `tools/activation-entry/activation_entry.py`
- `tests/test_m1_minimal.py`

缺少 Python 时不可用的能力：
- 本地编译 Blueprint 索引
- 本地运行最薄 route/query/entry 验证
- 本地执行 M1 的 unittest 回归

## 不受影响的能力

- `docs/` 文档体系
- `blueprint/` 节点内容
- snapshot/handoff 共享上下文

## 推荐降级策略

1. **首选**：安装 Python 3.10+，按现有脚本直接运行。
2. **无 Python 环境**：
   - 仅消费文档与 blueprint 节点（不跑本地编译与测试）；
   - 通过团队内已有 Python 环境的 CI/同伴执行验证；
   - 后续可补一个 Node 版本壳层作为可选运行面。

## 依赖边界说明

M1 当前是“最薄可运行骨架”实现，技术栈未被冻结为 Python-only。后续里程碑可在不改变协议语义的前提下增加其他实现语言。
