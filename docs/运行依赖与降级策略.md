# 运行依赖与降级策略

## 结论

当前仓库已切换到 **Go 模式B（二进制 runner + MCP server）**。

- 使用预编译二进制时，目标机器不需要 Python 或 Node 运行时。
- 开发态默认使用 Go 工具链执行 `go run` / `go test`。

## 影响范围

当前依赖 Go 的内容：
- `cmd/agent-pack-mcp/main.go`
- `internal/compiler/compiler.go`
- `internal/query/query.go`
- `internal/activation/activation.go`
- `tests/m1_minimal_test.go`

缺少 Go（仅开发态）时不可用的能力：
- 本地编译 Blueprint 索引
- 本地运行最薄 route/query/entry 验证
- 本地执行 M1 的 Go 回归测试

缺少 Go（发布态）但已有预编译二进制时：
- 仍可运行 compiler/query/entry 与 MCP server。

## 不受影响的能力

- `docs/` 文档体系
- `blueprint/` 节点内容
- snapshot/handoff 共享上下文

## 推荐降级策略

1. **首选（开发态）**：安装 Go 1.25+，直接运行：
   - `go run ./cmd/agent-pack-mcp compile ...`
   - `go test ./...`
2. **无 Go 环境（发布态）**：
   - 使用项目发布的预编译二进制 `agent-pack-mcp`；
   - 通过 MCP command 方式接入，不引入 submodule。
3. **仅文档消费**：
   - 只读取 `docs/` 与 `blueprint/`，不执行本地编译与验证。

## 依赖边界说明

M1 当前是“最薄可运行骨架”的 Go 实现，协议语义与 MCP surface 仍保持冻结优先。后续里程碑可在不改变协议语义前提下增加其他实现语言，但主分发面应保持模式B（二进制 runner）。
