# 00 总索引与使用说明

## 文档定位
这是一套面向 **Agent Pack 首期实现** 的拆分文档包。它不是再次解释总纲，而是把总纲重组为可独立派发、可并行阅读、可顺序推进的开发文档体系。

## 分类方式
本包只保留两类文档：

1. **开发里程碑开发指导文档**
   - 作用：告诉某个里程碑的执行 agent 这一步要做什么、按什么顺序做、交付什么、做到什么算完成。
2. **单里程碑所需上下文文档**
   - 作用：只给该里程碑完成任务所必需的上下文、契约、边界、模板和判断规则。
   - 原则：只提供“必须知道的事实”，不提供与本里程碑无关的泛化材料。

- M4 agent：`00 + 01 + 02 + 03 + 50 + 51 + 52`

## 强制使用方式
1. 每个里程碑 agent 只允许读取自己的最小阅读集。
2. 除非当前文档显式要求，否则不要回看全量总纲。
3. 当前里程碑只消费前置里程碑的**交付物**，不重新解释前置里程碑的设计动机。
4. 当前里程碑交付完成后，必须向下一里程碑输出：
   - 固定交付物清单
   - 已冻结对象清单
   - 未解决风险清单
   - 明确不可越界事项
5. 所有里程碑都必须同时更新：
   - `bd issue` 状态
   - `Context Snapshot`
   - `Handoff`

## 统一输出格式要求
每个里程碑完成时都要产生一份简短的 handoff 说明，包含：
- 本里程碑已完成什么
- 哪些对象已经冻结，后续不可随意改
- 下一里程碑可以依赖什么
- 下一里程碑绝对不要做什么

同时建议至少保留一份 `Context Snapshot`，记录：
- 当前事实状态
- 输入/输出对象
- 风险与阻塞
- 后续待决问题

## 整体推进顺序
M0 -> M1 -> M2 -> M3 -> M4

任何跳过顺序的实现，都视为高风险偏航。
