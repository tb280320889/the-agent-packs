# 00 总索引与使用说明

## 文档类型
总索引 / 使用说明

## 作用域
这是一套 **Agent Pack 正式文档包**。它的职责不是继续讨论原则，而是把总纲、系统层母版、领域树母版、首期实现顺序，以及多-agent 协作规则，重组为一套可以独立派发给不同开发 agent 的文档体系。

## 这版相对 v1 的关键修正
1. 把 **“渐进式披露系统层设计”** 提升为前置主文档，而不是散落在里程碑上下文里。
2. 把 **Git truth layer -> Markdown + frontmatter -> SQLite index -> Blueprint Query MCP -> 最小上下文包** 固定为系统骨架。
3. 明确区分：
   - **长期 operating model 母版**
   - **当前仓库首期实现切片**
4. 把 v1 中补齐但尚未一级化的对象正式收口：
   - Activation Result
   - Validation Plan
   - Route 冲突裁决与退回规则
   - 全局 schema 与 pack 局部 contract 的边界
   - Breaking change 治理
   - Golden / negative / partial / handoff 测试矩阵

## 文档分类
本包只有两类文档：

### A. 开发里程碑开发指导文档
作用：告诉该里程碑 agent 这一步 **做什么、按什么顺序做、交付什么、做到什么算完成**。

### B. 单里程碑所需上下文文档
作用：只给该里程碑完成任务所必须知道的上下文、契约、模板、判断规则。  
原则：**不重复、不冗余、不遗漏，不把与本里程碑无关的泛化材料塞进去。**

## 强制使用方式
1. 每个里程碑 agent 只读自己的 **最小阅读集**。
2. 当前里程碑只消费前置里程碑的 **冻结对象与交付物**，不重新发明前置设计。
3. 如需超出最小阅读集，必须先判断是否属于当前文档作用域；不是，就不读。
4. 任何里程碑都不允许绕过 `02`。这是系统层母版。
5. `03` 不是“当前就要全做”，而是保证长期 operating model 不被首期切片误伤。
6. `04` 与 `05` 是共享协作层，不属于可选材料；任何接力开发都必须遵守。

## 改造计划 v1 的使用方式
- `docs/改造计划v1/` 是**增量改造文档包**，用于后续 session 在明确 bead 或开发需求指向“分层索引 / package 命名治理 / 文档吸收管道 / 迁移演练”时再进入。
- 默认 agent **不要**因为看到该目录就全量阅读；只有当 bead 或 handoff 明确要求时，才按 `docs/改造计划v1/00-总索引与使用说明.md` 中的最小阅读集进入。
- `docs/改造计划v1/` 不替代当前 `docs/00~52` 正式主线；它是后续改造工作的受控入口，不是新的默认总纲。

## 统一交付接口
每个里程碑完成时，都必须向下游交付一份 handoff 摘要，至少包含：
- 本里程碑完成了什么
- 哪些对象已经冻结
- 下一里程碑可以依赖什么
- 下一里程碑不得修改什么
- 仍存在哪些风险，但它们不阻塞下一里程碑

同时必须更新：
- `bd issue` 状态
- `Context Snapshot`
- `Handoff`

**长期 operating model 可以大，但当前开发切片必须小；系统层母版必须完整，首期实现面必须收敛。**
