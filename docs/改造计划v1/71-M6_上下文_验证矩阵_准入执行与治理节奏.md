# 71 M6 上下文：验证矩阵、准入执行与治理节奏

## 文档类型
单里程碑所需上下文文档

## 一、M6 运行验证最小矩阵
- 注册表验证：字段、样本、reserved、alias、canonical name。
- route 验证：global -> domain -> capability attach 顺序。
- 资产验证：`source_uri -> target_scope -> target_ref -> 落位`。
- validator 验证：最小上下文包消费、负例拒绝、handoff 输出完整性。
- 第二领域线验证：主域接入、capability attach、正式 package 准入、负例阻断。

## 二、正式准入执行条件
- M5 已无关键 `blocked` 整改项。
- `remove-before-m6` 兼容项已清退。
- 第二领域线的注册、资产、route、validator、handoff 链路都已有可重复验证结果。

## 三、治理节奏
- 每次新增领域线前先检查注册表与命名空间。
- 每次吸收新资料前先检查 M3 映射与落位规则。
- 每次新增 package 或 capability 前先通过准入检查表。
- 每次重大迁移后补 snapshot、handoff 与 bead 状态。

## 四、M6 验收结果类型
- `ready-for-adoption`：可进入正式复用与扩域。
- `ready-with-guardrails`：可进入有限复用，但仍需治理观察。
- `blocked`：仍有关键链路未通过。
- `reject`：当前准入路径违反冻结边界，不得推进。

## 五、禁止事项
- 只验证成功路径，不验证负例与回滚。
- 让治理节奏依赖单个 agent 的隐性记忆。
- 在无 bead、snapshot、handoff 的情况下宣布 v1 可规模化复用。
