# 31 M2 上下文：命名规则、注册表字段与冲突裁决

## 文档类型
单里程碑所需上下文文档

## 一、命名规则

### 主域编排包
`<domain>-orchestrator`

### 主域 workflow package
`<domain>-<subdomain>`

### 横线 workflow package
`<capability-line>-<subdomain>`

## 二、禁止的裸名
以下名称默认禁止直接作为 package 名称：
- `release`
- `security`
- `auth`
- `payment`
- `manifest`
- `review`

## 三、注册表最小字段
- `name`
- `kind`
- `domain`
- `subdomain`
- `category`（orchestrator / workflow / capability）
- `canonical_blueprint_node`
- `visibility_scope`
- `activation_mode`
- `aliases`
- `reserved`

## 四、冲突裁决规则
当新增 package 时必须检查：
1. 名称是否重名
2. alias 是否撞已有 canonical name
3. 是否错误使用裸名
4. 是否误把 capability package 放进主域主竞争空间
5. 是否应复用已有 capability line 而非新建全局入口
