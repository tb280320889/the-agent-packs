# 00 总索引与使用说明

## 文档类型
总索引 / 使用说明

## 作用域
本目录用于承载 **改造计划 v1** 的独立文档包。

它不替代当前 `docs/00~52` 正式主线文档，而是回答下面这个问题：

**在不破坏已冻结 Phase 1 壳层的前提下，如何把当前“最薄闭环实现”升级为支持多主域、多横线、多 workflow package 持续扩张的可维护骨架。**

## 这套文档解决什么
1. 如何把“项目内部维护 agent / 迭代开发 agent / 文档吸收 agent / 用户侧外部 agent”角色体系正式化。
2. 如何把当前较平铺的路由实现，升级为“总编排 -> 主域编排 -> 子领域/横线挂接”的分层索引模型。
3. 如何治理 `release`、`security`、`auth`、`payment` 一类高复用横线名称，避免后续 workflow package 增长时冲突与误路由。
4. 如何在不破坏现有 M0~M4 冻结对象的前提下，为第二领域线和后续领域线接入打基础。
5. 当 M4 只完成“计划与验收边界冻结”而未完成真实实现迁移时，如何继续扩展 M5、M6 等后续执行里程碑，把 v1 从文档骨架推进到稳定运行状态。

## 当前状态
- `docs/改造计划v1/` 已成为当前唯一有效的开发文档包。
- 旧的 `docs/00~52`、`docs/context-snapshots/`、`docs/handoffs/` 仅在删除前用于迁移校对，不再作为新 session 的入口或共享上下文来源。

## 进入条件
当前开发默认进入本目录，并按 GSD 任务项选择最小阅读集。

## 使用原则
- 本目录是**按 GSD 任务项驱动**的增量文档包，不是新的全量主文档集合。
- 新 session 的 agent 不应因为目录存在就全读；只读与当前 GSD 任务项、handoff、开发需求直接相关的最小阅读集。
- 若当前任务只需实现某一个改造子目标，应只读对应里程碑开发指导与最少上下文，不回退为全目录阅读。

## 文档分类
本目录沿用正式文档包的组织方式，分两类：

### A. 开发里程碑开发指导文档
用于定义某一改造里程碑：
- 做什么
- 不做什么
- 顺序是什么
- 交付什么
- 什么时候算完成

### B. 单里程碑所需上下文文档
用于定义该里程碑最小必要上下文：
- 结构模型
- 约束边界
- 命名规则
- route 规则
- 交接规则

## 统一交付接口
每个里程碑完成时必须留下：
- 对应 GSD 任务项状态更新
- 一份 Context Snapshot
- 一份 Handoff
- 已冻结对象清单
- 风险与未决项

## 当前推荐启动方式
新 session 的 agent 推荐按以下顺序进入：
1. 先读 `docs/改造计划v1/00-总索引与使用说明.md`。
2. 再只读当前 GSD 任务项所属里程碑的最小阅读集。
3. 再读取对应 handoff 与 snapshot。
4. 结合 GSD 任务项执行，不补读无关阶段文档。

## 一条必须始终记住的话
**改造计划 v1 的目标不是推翻现有正式主线，而是把现有“最薄闭环”升级为“可持续扩域且不乱名、不乱路由、不乱吸收知识”的可复制骨架。**
