# 改造计划 v1 Session 进入说明

## 作用
本文件用于给新 session 的 agent 一个**最小进入路径**，避免其因为看到 `docs/改造计划v1/` 就全量阅读整个目录。

## 进入规则
只有当当前 bead 或 handoff 明确属于改造计划 v1 时，才进入本目录。

## 最小启动顺序
1. 先读 `docs/改造计划v1/00-总索引与使用说明.md`
2. 再读当前 bead 所属里程碑的最小阅读集
3. 读取对应 snapshot / handoff
4. claim 当前 bead 后开始执行

## 按需求映射阅读入口

### 若需求是角色与边界收敛
读：`10 + 11 + 12`

### 若需求是总编排 / 分层索引 / 子编排
读：`20 + 21 + 22`

### 若需求是 workflow package 命名冲突 / 注册表 / 保留名
读：`30 + 31`

### 若需求是文档吸收 / 外部资料纳入 / 资产化
读：`40 + 41`

### 若需求是迁移步骤 / 演练 / 回滚 / 验收
读：`50 + 51`

## 禁止事项
- 不要把 `docs/改造计划v1/` 当作新的默认总纲全量阅读
- 不要在未 claim bead 的情况下预先展开多个改造方向
- 不要因为文档目录完整，就默认所有后续 bead 已经规划完成

## 当前状态
- 当前目录只是改造计划文档包与交接入口，后续 bead 仍应按真实开发需求增量创建。
