# 40 M3 文档吸收管道与知识资产化 开发指导

## 文档类型
开发里程碑开发指导文档

## 作用域
M3 负责把“文档型 agent”从口头设想，升级为可执行的外部资料吸收管道。

## 本里程碑目标
1. 定义外部资料候选池
2. 定义 skill / MCP / llm.txt / best practice 的吸收流程
3. 定义文档型 agent 的落位职责
4. 定义外部资料如何映射回 domain / capability / package 空间

## 本里程碑输入边界
- 只消费共享文档 `00~05`、M2 已冻结的注册表与命名治理输出、当前 M3 文档。
- 默认以上游已注册命名空间为落位前提，直接消费 `workflow-packages/registry.json` 中的 `domain / capability / package` 身份判断。
- 外部资料只能作为候选知识源，不得反向定义顶层 envelope、路由层级、注册表字段或命名规则。
- 不为某一份外部资料单独放宽注册表、attach-only 或候选空间规则。

## 必须产出
1. 资料吸收流程
2. 语义映射规则
3. 资产落位规则
4. 给 M4 的迁移与演练建议

## 本里程碑建议直接落地的共享产物
- M3 阶段快照：`docs/改造计划v1/context-snapshots/<date>-m3-knowledge-ingestion.md`
- M3 交接文档：`docs/改造计划v1/handoffs/<bead-id>-handoff.md`
- M3 输出摘要：在文档中明确“候选池 / 语义映射 / 审查闸口 / 资产落位 / backlog 处理”，避免 M4 再次回头补治理约束。

## 设计顺序
1. 先冻结可接收资料类型与候选元数据。
2. 再冻结“候选收集 -> 去重归类 -> 规范映射 -> 冻结审查 -> 资产落位”的标准流程。
3. 再冻结系统层、主域、横线能力、package 专属资料的落位规则。
4. 最后定义 backlog、拒收、迁移与验收接口。

## M3 必须回答的问题
- 一份外部资料进入仓库前，最小必要元数据是什么，谁来补齐，缺失时如何处理。
- 文档型 agent 如何把外部资料判断为 `system / domain / capability / package / reject`。
- 一份资料如何绑定到已注册命名空间，避免生成“有内容但无归属”的游离资产。
- 什么时候形成正式资产，什么时候只进入 backlog，什么时候必须拒收。
- M4 在迁移与准入演练时，如何验证一份资产既可追溯来源，又不突破 M2 的命名边界。

## 明确不产出
- 不新增未注册的主域、capability line 或 package 名称。
- 不把外部资料原样复制进 Blueprint 正文作为主路由依据。
- 不让文档型 agent 绕过注册表直接生成 workflow package。
- 不在本里程碑重写 route / validator / activation 顶层协议。

## 完成标准
1. 外部资料不再直接散落入仓。
2. 文档型 agent 有固定工作流。
3. 外部资料可以被有边界地纳入现有知识图谱。
4. 所有正式资产都能追溯到已注册的 `system / domain / capability / package` 命名空间。
5. M4 可以直接消费 M3 的吸收结果做迁移与准入演练。

## 本阶段建议冻结的最小对象
- 候选资料元数据最小集合：`source_type`、`source_uri`、`captured_at`、`owner`、`trust_level`、`target_scope`、`target_ref`、`disposition`。
- 处理状态最小集合：`candidate`、`mapped`、`frozen`、`backlog`、`rejected`。
- 审查闸口最小集合：命名空间校验、语义去重、非目标检查、落位检查、可消费性检查。

## 文档型 agent 的工作流要求
- 先收集候选，不直接写主文档。
- 先做语义映射，不直接做正文合并。
- 先通过注册表归属校验，再决定落位目录。
- 若无法映射到已注册命名空间，则必须进入 backlog 或 reject，不得以临时文档规避。
- 交付时必须同步留下 bead 状态、Context Snapshot、Handoff 与风险说明。

## 给后续里程碑的最小输出要求
- 对 M4：迁移与准入演练必须检查资产来源、命名空间归属、落位正确性、是否仍可被 route/read/bundle 间接消费。
- 对后续新增领域线：先扩展注册表，再允许 M3 管道吸收该领域外部资料；禁止反向用资料先占坑再补注册表。
