# 10 M0 角色冻结与边界校正 开发指导

## 文档类型
开发里程碑开发指导文档

## 作用域
M0 只做一件事：**把改造计划 v1 的角色体系、系统边界、冻结面与不可动面先钉死。**

## 本里程碑目标
1. 冻结内部维护 agent / 迭代开发子 agent / 文档型 agent / 用户侧外部 agent 的角色模型
2. 冻结“总编排先于主域编排，主域编排先于横线挂接”的系统判断
3. 明确现有正式主线哪些对象完全不动
4. 明确改造计划 v1 只改结构，不改现有顶层 envelope 语义

2. 不可修改对象清单

## 明确不产出
- 具体 query 重构实现
- package 注册表实现
- 文档吸收流水线实现

## 完成标准
1. 后续里程碑不再争论角色定义
2. 后续里程碑知道哪些对象不能碰
3. M1 可以直接在冻结边界上设计分层索引模型
