# 51 M4 上下文：迁移步骤、回滚策略与验收清单

## 文档类型
单里程碑所需上下文文档

## 一、建议迁移顺序
1. 先冻结角色模型与边界
2. 再冻结节点分类与 route 分层规则
3. 再设计 package 注册表与命名治理
4. 再设计文档吸收管道
5. 最后做实现迁移与第二领域线演练

## 二、回滚策略
- 若节点分类设计导致既有 route 语义不清，回滚到“仅文档冻结，不落实现”状态
- 若注册表设计无法兼容现有 package，回滚到“先补 registry 草案，不执行迁移”状态
- 若外部资料吸收规则过重，回滚到“候选池 + 手工映射”最小模式

## 三、验收清单
1. 是否已有明确的总编排 / 主域编排 / 横线挂接三层模型
2. 是否已有 package 注册表字段与命名规则
3. 是否已禁止裸名 package
4. 是否已有文档吸收标准管道
5. 是否已有第二领域线接入前的演练方案
