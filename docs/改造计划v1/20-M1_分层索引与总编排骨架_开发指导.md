# 20 M1 分层索引与总编排骨架 开发指导

## 文档类型
开发里程碑开发指导文档

## 作用域
M1 负责把“平铺路由”升级为“总编排 -> 主域编排 -> 横线挂接”的结构设计骨架。

## 本里程碑目标
1. 定义节点分类与可见性
2. 定义 global orchestrator 与 domain orchestrator
3. 定义两段式 route 流程
4. 定义横线 attach-only 规则

## 必须产出
1. 节点分类清单
2. 可见性规则
3. 两段式 route 规则
4. 给 M2 的“注册表字段需求”摘要

## 本里程碑建议直接落地的共享产物
- M1 阶段快照：`docs/改造计划v1/context-snapshots/<date>-m1-layered-routing.md`
- M1 交接文档：`docs/改造计划v1/handoffs/<bead-id>-handoff.md`
- M2 输入摘要：在 M1 文档中明确写出注册表至少需要承载哪些字段，避免 M2 再回头猜测

## 明确不产出
- package 注册表实现细节
- 外部资料吸收流程

## 完成标准
1. 后续 package 新增时知道应该挂在哪个候选空间
2. 横线包不再被视作第一轮主竞争候选
3. M2 可以直接据此设计注册表

## 给 M2 的最小输入要求
M1 完成后，至少要给 M2 留下以下判断结果：
- 候选空间至少分为 `global / domain / capability` 三类
- 节点需要可表达 `node_kind / visibility_scope / activation_mode`
- 注册表必须能判断一个 package 是主域 workflow 还是横线 capability
- `attach-only` 语义必须可被注册表和 route 同时消费，而不是只写在文档里
