# 99 融合映射计划表

## 作用域
本文记录从旧版 `docs/` 与 `docs/v2/` 融合为唯一正式文档包的映射方案。  
本次采用 **方案 A**：以原 `docs/v2/` 为新的系统与里程碑主线，把现有协作层文档并入正式编号体系。

## 一、融合原则
1. 原 `docs/v2/` 中关于系统层、里程碑主线、MCP、SQLite、Compiler、workflow package 的定义优先。
2. 现有 `docs/02`、`docs/03` 保留其内容价值，但后移编号，成为系统主文档之后的协作层主文档。
3. 最终仓库只保留一套正式文档路径：`docs/*.md`。
4. 原 `docs/v2/` 在完成迁移后删除，避免双真相源。

## 三、术语统一策略
- 正式名称使用 `workflow package`，在需要照顾旧语境时可注明“简称 pack”。
- 正式名称使用 `Blueprint Query MCP`，简称可写 `Query MCP`，但首次出现必须写全。
- 正式名称使用 `最小上下文包`，可在字段名或英文示例中保留 `minimal context bundle`。
- 正式名称使用 `总编排`，需要英文标识时写 `global orchestrator`。
- 正式名称使用 `横线`，必要时说明对应英文概念为 `cross-cutting line`。

3. `AGENTS.md` 只指向新的正式编号体系。
4. 所有 README / 模板引用都更新到正式路径。
