# 11 M0 上下文：系统定义与硬边界

## 文档类型
单里程碑所需上下文文档

## 作用域
本文件只给 M0 提供 **项目定义、运行原则、硬边界、禁止事项**。  
它不定义字段级 schema，也不定义具体 pack 细节。

## 本文必须回答的问题
1. 这个项目本质上是什么。
2. 它不是什么。
3. 没有 activation request 时系统该不该运行。
4. agent-pack 能拿什么上下文，不能拿什么上下文。
5. 为什么它不能变成业务仓、扫描器或自治平台。

---

## 一、项目定义

项目本质应固定为：

**由外部调用者 agent 显式激活的、面向宿主 monorepo 的、非侵入式增强能力包系统。**

这个定义有 5 个必须长期保持不变的点：

1. **由外部调用者 agent 显式激活**  
   没有 activation request，就不进入领域执行逻辑。

2. **面向宿主 monorepo**  
   它服务的是外部宿主仓库中的任务，而不是本仓库中的业务代码。

3. **非侵入式**  
   不要求宿主改目录，不要求迁移代码，不要求统一脚手架。

4. **增强能力包系统**  
   它提供的是 domain packs、skills、validators、contracts、blueprint knowledge、MCP resources / prompts / tools、artifact templates。

5. **通过 agent 服务宿主**  
   它增强的是调用它的 agent，而不是直接接管宿主仓库。

---

## 二、明确的非目标

### 不是业务工程
本仓库不是具体业务产品主仓。

### 不是产品模板仓
它不输出一套统一业务模板让所有宿主硬套。

### 不是自动扫描器
它不默认整仓扫描，不默认模糊发现，不默认递归推理所有领域。

### 不是自治分析平台
没有 activation request 时，它不主动进入领域分析或执行。

### 不是“大而全工具平台”
工具存在是为了服务 route / bundle / validator / handoff 闭环，不是为了堆功能面。

---

## 三、边界化上下文原则

agent-pack 只能使用调用者明确提供的 **bounded context**。  
推荐的上下文形式：
- selected files
- config fragments
- path hints
- host hints
- browser hints
- bounded repo excerpts

### 明确不推荐
- 整仓扫描
- 默认递归推理所有领域
- 因上下文不足就编造宿主事实

### 上下文不足时必须怎么做
返回：
- 当前上下文不足
- 建议补什么
- 当前能安全得出的部分结论

而不是继续猜。

---

## 四、三块核心系统模型

### 1. Blueprint Content System
它定义宿主无关的增强知识。  
本质是：**结构化、可路由、可组合的知识节点系统。**

### 2. Blueprint Index & Query System
它把 Blueprint 节点编译成可查询、可路由、可裁剪的增强图谱。  
本质不是全文搜索引擎，而是：**路由与上下文构建层。**

### 3. Agent Consumption System
它定义调用者 agent 如何激活并消费增强能力。  
本质不是宿主管理器，而是：**agent 面向 enhancement pack 的消费控制层。**

## 五、必须长期稳定的核心对象名
以下对象名在项目中应尽量长期稳定，不允许同义乱换：

- Agent Enhancement Pack
- Activation Request
- Routing Result
- Context Bundle
- Artifact
- Validation Plan
- Validator Result
- Activation Result
- Handoff Bundle
- Blueprint Node
- Workflow Package
- Blueprint Query MCP

---

## 六、M0 必须守住的硬边界

### 1. 不能让系统自启动
没有 activation request 时，不能进入领域执行逻辑。

### 2. 不能让系统默认拿无限上下文
bounded context 是强边界，不是建议。

### 3. 不能让 Blueprint 退化成普通文档库
必须保留结构化、可路由、可组合这三个属性。

### 4. 不能让 agent-pack 侵入宿主
宿主可以被服务，但不应被强迫改造。

### 5. 不能让 M0 变成实现层
M0 只冻结公共壳层。

---

## 七、哪些判断后续里程碑都不能推翻
1. agent-driven 显式激活
2. bounded context
3. 非侵入式
4. enhancement pack system
5. Blueprint 不是自由文档集
6. 项目不是自治平台
